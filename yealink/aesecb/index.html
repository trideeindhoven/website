AES ECB encryption

Yealink has decided to use Advanced Encryption Standard Electronic codebook (AES-ECB) encryption.
While AES encryption is considered to be same, the mode chosen by Yealink can be categorised as…sub par. ECB mode encryption is considered to be a flawed type of encryption because of the “ECB penguin” attack. I will explain this in a visual example:

First of all: “Tux” is the mascotte of the operating system Linux. It is a drawing of a penguin:

When performing AES-ECB encryption on the above image, it is easy to see the ECB mode is a highly suboptimal encryption method:

Now, for text it may still not be possible to recover the original text (yet), but why choose an encryption mode that is known to be flawed for almost two decades. The first time ECB Tux was posted on Wikipedia was in 2006 so this is hardly a new attack.
